# Привет, новый участник сообщества!

Если ты решил стать частью нашего проекта, чему мы несказанно рады, прочитай пожалуйста данную статью о том как устроен репозиторий и как добавить в него свое решение

Прежде всего ты должен убедиться в том, что данного решения нет или что оно недостаточно: лучше потратить силы на задачи, которые ещё не были решены. В случае, если ты всё же хочешь добавить решения которое уже есть, тебе никто не помешает. 

## О технологии Git
Данный проект реализован на технолигии git и размещён на github. Этот выбор обоснован в первую очередь, возможностью одновременного редактирования проекта несколькими людьми одновременно. На данный момент возможно обновление репозитория только по данной инструкции.

## Оформление

Все файлы в репозитории должны быть оформленны в одном стиле для понятной и простой навигиции, а также для возможности автоматизации, т. е. запуска скриптов, телеграм-ботов и прочих приколов. 

Все материалы располагаются по адресу subjects/ПРЕДМЕТ/ПРЕПОДАВАТЕЛЬ/

Внутри материалы могут быть распределены на папки lectures, labs, practice и exams. Если предмет по своемму устройству отличается, действуй на своё усмотрение, но отмечай это в коментариях.

Далее должна лежать сама папка с заданием и только в ней твой вариант, например:
```
.../Labs/Lab_1/Lab_1_by_Anton_K
```

Также не забудь добавь изменения в главный README в разделе Subjects и teachers.md, если таких решений ещё не было

*По содержанию ни в коем случае нельзя допускать попадания чужой личной информации!*


Также крайне не рекомендуется добавлять файлы размером > 10 MB

## Как добавить решение

Итак, ты решил задание. Что дальше? Ну для начала необходимо установить git на свой ПК, если ты этого ещё не сделал.

### Windows
Для работы на windows перейди по ссылке <https://gitforwindows.org/> и следуй инструкции.
### Mac
На устройствах от Apple git установлен по умолчанию для этого открой терминал и введи команду
```bash
git --version
```
### Linux
```bash
sudo apt install git-all
```
---
Все остальные команды ты можешь выполнять как в командной строке, так и в расширениях твоего IDE.

---

### Создай fork оригинального репозитория

На сайте проекта, немного правее назввания нажми на кнопку fork

### Клонируй fork на свой ПК
```bash
git clone https://github.com/ТВОЙ_ЛОГИН/BSUIR-HOMEWORK.git
```

### Создай новую ветку для решения
```bash
git checkout -b add-oaip-lab-1
```

### Добавляй своё решение и делай commit
```bash
git add subjects/FAP/krivonisiva/Labs/Lab1
git commit -m "Добавил решение 1 лабы по ОАиП"
```

### push на сервер
```bash
git push origin add-oaip-lab-1
```

### Создай pull request
На сайте github создаём pull request с оригинальным репозиторием в ветку staging. Теперь твои изменения должен одобрить другой участник сообщества что бы они вступили в силу.

#### Жди
Твой PR могут отклонить. В комментарии будут указаны причины, не переживай всё можно исправить, как только ты снова сделаешь push проекта, твой PR обновится и снова будет ожидать подтверждения.
